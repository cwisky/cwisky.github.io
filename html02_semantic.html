HTML5 시맨틱 태그 정리

HTML5에서는 문서 구조를 더 명확하게 표현하고 접근성과 SEO를 향상시키기 위해 다양한 시맨틱 태그(Semantic Tags)가 추가되었습니다. 이러한 태그들은 단순히 시각적인 레이아웃이 아니라 의미 있는 구조를 제공하기 위해 도입되었습니다.

✅ 시맨틱 태그란?

"Semantic"은 "의미론적인"이라는 뜻으로, 시맨틱 태그는 태그 자체가 어떤 의미나 역할을 내포하고 있어 코드의 가독성과 유지보수성을 높입니다.

📚 주요 시맨틱 태그와 설명

태그 설명
<header>문서나 섹션의 머리말(헤더)을 정의
<nav>내비게이션 링크 영역을 정의
<main>문서의 주요 콘텐츠를 정의
<section>주제를 구분하는 독립적인 섹션을 정의
<article>독립적으로 배포 가능한 콘텐츠를 정의
<aside>본문과 간접적으로 관련된 보조 콘텐츠를 정의
<footer>문서나 섹션의 꼬리말(푸터)을 정의
<figure>이미지, 다이어그램 등의 콘텐츠와 설명을 묶는 태그
<figcaption><figure>에 대한 설명을 제공

💻 예제: 시맨틱 태그를 사용한 간단한 웹 페이지 구조

<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <title>시맨틱 태그 예제</title>
</head>
<body>

    <header>
        <h1>나의 블로그</h1>
        <p>프론트엔드 개발 이야기</p>
    </header>

    <nav>
        <ul>
            <li><a href="#">홈</a></li>
            <li><a href="#">소개</a></li>
            <li><a href="#">글 목록</a></li>
            <li><a href="#">연락처</a></li>
        </ul>
    </nav>

    <main>
        <section>
            <h2>최근 게시물</h2>
            <article>
                <h3>HTML5란 무엇인가?</h3>
                <p>HTML5는 웹의 최신 표준입니다...</p>
            </article>
            <article>
                <h3>시맨틱 태그의 중요성</h3>
                <p>의미 있는 구조는 접근성을 높여줍니다...</p>
            </article>
        </section>

        <aside>
            <h2>광고</h2>
            <p>이 자리는 광고입니다.</p>
        </aside>
    </main>

    <footer>
        <p>&copy; 2025 나의 블로그. All rights reserved.</p>
    </footer>

</body>
</html>

📝 정리

  • 시맨틱 태그는 문서의 구조를 명확하게 표현하여 개발자, 브라우저, 검색 엔진 모두에게 이점을 줍니다.
  • <div>나 <span> 대신 의미 있는 시맨틱 태그 사용을 권장합니다.
  • 접근성(Accessibility)과 SEO(Search Engine Optimization)를 고려할 때 매우 중요합니다.
